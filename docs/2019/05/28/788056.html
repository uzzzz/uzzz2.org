解决Pyshark无法抓包问题

Python+Pyshark（基于Wireshark的wrapper）可以抓取并分析网络包，之前的测试机一直都可以正常抓取，后来重新换了一台测试机，发现死活也抓不了，永远是零个包，如下所示。

import pyshark
cap = pyshark.LiveCapture()
cap.sniff(timeout=1)
cap
<LiveCapture (0 packets)>

百思不得其解， 于是之后一顿百度和Google, 终于找到一篇文章，里面说他也遇到了类似的问题，才得以解决。
https://www.cnblogs.com/lsdb/p/9765136.html

问题定位及fix

当出现无法抓包的问题时，可以先使用set_debug()使能debug功能，然后你在sniff的时候就会打印调试信息，这里我们看到报错信息“>>> tshark: You can’t write both raw packet data and dissected packets to the standard output.”， 此现象和上面文章所描述一致。

补充说明

Python我使用的时Python2.7, 所以安装pyshark时装的是 pyshark-leagacy（pip install pyshark-legacy）, 版本是0.3.8，因为后面的版本支持python3了。
此外，当python2和python3都安装的情况下，可以用"py -2"和"py -3"来指定使用python2或python3. 特别的，如果要使用python2的pip程序，可以用"py -2 -m pip install xxxx"。
