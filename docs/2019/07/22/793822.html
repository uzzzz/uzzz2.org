程序员，请从25岁开始打造个人品牌

  • 柚子社区
  • Donate

周末的时候，带儿子去书店看书，无意中看到一本书，书名为《25岁，如何规划你的人生》，作者是日本人气作家松浦弥太郎。我读过好几本松浦弥太郎的书，本公众号就是在读了他的《给40岁的崭新开始》之后受到启发而创建的。

其实，我早过了读这本书的年龄，不过出于对人生进行复盘的心态，我还是仔细阅读了这本书，希望了解我在25岁没做好的事情，才导致现在一事无成。这是一本小册子，颇有心灵鸡汤的味道，列出了50个人生要点。这里不详细讨论这50点人生要义，有兴趣的可以上网搜一搜，或者去书店翻一翻。其中有两点比较触动我：打造自己专属的发声平台 和 建立自我品牌，迎来更多机会， 在我看来，其实表达了同一个意思，那就是打造个人品牌。

打造个人品牌，并不是说要去创建一个商业品牌，比如像李宁那样，而是建立一个人设。为什么程序员需要打造个人品牌呢？这要从大龄程序员的出路谈起，关于大龄程序员的话题，之前已经写过两篇文章：

  1. 为啥你身边没有40+的程序员

  2. 明明拼智力的编程，为何还要拼年龄？

我个人观点也非常明确，年龄并不会成为程序员的阻碍，年龄大了，照样可以编程。但是，可以做并不代表这条路就那么容易走。比如你过了35岁，还像25岁那样去投递简历，大概率的会被HR扔到垃圾桶，哦，不对，是扔到为垃圾邮箱中。是年龄歧视吗？也不尽然。这就如同，大学生去劳务市场，也大概率找不到工作，这时学历其实是一个减分项。很多公司招聘条件上限制35岁以下，并不表示这个岗位过了35岁就做不了，而是有句潜台词：这个人都过了35岁，还以这样的方式找工作，那一定很水吧，找来面试，也是浪费大家的时间，还是算了吧。

谈到大龄程序员失业，大部分人可能会认为是因为人到中年，不用功、无法跟上新技术，水平差，才导致失业的。前段时间甲骨文大裁员，一堆人在那冷嘲热讽。其实，大多数人被裁，是因为被一锅端了。啥意思呢？就是公司因为方向调整，砍掉某个产品线，那这个产品线的人，从上到下，不管你是优秀员工，还是打酱油的，一律被干掉。我自己亲眼目睹过的两次裁员，都是这样。所以以后碰到被裁员的员工，不要给别人贴上水平差、混日子的标签。虽然在裁员面前人人平等，但不同人的反应会不一样。年轻的、水平高的，会高高兴兴的拿着补偿，然后找下家。最尴尬的就是那种工作了多年的资深员工，虽然拿到一笔不菲的补偿，但这点钱也只够支撑个一年半载的，如果还背着房贷的，心就更慌了，还是得再去找份工作。

这样说来，大龄程序员真是有点晚景凄凉。但别慌，我并不是那种悲观主义者，下面就说说大龄程序员找工作的正确姿势。一般来说，大龄程序员能找到工作的方式有：1. 主动邀请；2. 猎头推荐；3. 内推。除此之外，也有可能，但那会无比艰难，甚至有可能信心会被打击没了。

虽然说，酒香不怕巷子深，但对于大多数人而言，虽然水平不差，但还没有香到那种程度，能够让伯乐自动寻上门来。这个时候，就需要本文所探讨的主题，打造个人品牌。一般而言，最常用的手段就是找同学、同事、前同事推荐。但如果你在同学（同事）心目中，不是一个靠谱的人，那别人会推荐吗？反之，如果你在同学、同事心目中就是一个技术水平高、做事认真、有担当的人，如果有机会，别人能够第一时间想到你，那推荐的可能性就更大。别小看这种推荐，它能极大减少招聘双方的成本，要知道，招聘成本也是很高的。

那么问题来了，程序员该如何打造个人品牌呢？通常可以采取如下手段：

写文章、做演讲

对于程序员而言，写技术文章是最自然，也是最有效的一种手段。一方面可以帮自己梳理一下知识，另一方面也可以帮助他人，何乐不为呢？也许某次的面试官，正好拜读了你的文章，且收益颇多，那无形中是不是给你加了很多分？

有人可能会说，我做的东西都很基础，写出来会不会很low。这个不会，你看小学知识，都是好几百年前的知识，教材还一直在更新。Java入门，不断有人在写，但每年依然有书出版。简单不要紧，要记得，每年还有那么多新手加入编程大军。如果你能把一个简单的东西，讲得很透彻，也是一种本事啊。

还有人可能说，我做的东西比较冷门，估计写了，也没有人看。错，越是冷门的东西，越是有写的价值，只有这样，才能吸引到同行，否则在茫茫人海中，哪里去找志趣相投的人。互联网就是如此神奇，能把同类人拉的很近。早些年，我写过一些关于浏览器引擎研究的文章，这个在二线城市，基本没有市场，但却收到了腾讯、阿里、百度、华为这些大厂的邀约，因为只有这些大厂，才会有做浏览器引擎的需求（最后由于工作地点问题，没有去到这些大厂，略有遗憾）。

关于写作的平台，前几年首推博客。这几年，博客开始没落，但它比较容易被搜索引擎检索到，仍然是推荐平台。另外有更多的平台可以选择，微信公众号、知乎专栏、简书等，不用纠结，选择一个平台，动手就是成功的开始。当然，如果有精力，同时在几个平台上写也是可以的。

写技术文章，不太需要什么文笔，只要能把事情讲清楚。当然，能够将技术写的深入浅出，也需要一定的水平，如果你认为写文章不是自己的强项，还可以考虑用代码说话。

更多的人是兴致勃勃的开了博客（公众号、专栏），写了几篇文章，然后一看，没啥阅读量啊，于是就泄气了。这个时候你应该换一个角度想，你写文章是为了梳理自己掌握的知识，帮助自己提高，有没有人阅读，有那么重要么？

github

当年，python之父去Google面试，简历上就写了句：I wrote python，是不是霸气侧漏？作品是程序员最好的简历，当然我们大多数人没有这么牛X，但如果你在Github上有一两个比较火的开源项目，是不是也可以把链接甩给面试官，让面试官对你刮目相看。

还不知道github是什么？赶快面壁去。github号称是是世界上最大的程序员（同性）交友网站，也被人戏称为GayHub。关于github的介绍，我前几天看过一篇文章：Github是目前唯一的还有流量红利的写作平台，写的比较详细，有兴趣可以读一读。

大多数程序员只是把github当作软件仓库，丢一堆示例代码上面（这方面我也做的不好，主要是我之前没有打造个人品牌的意识），这样非常不妥。解决的方法可以建两个账号，一个作为自己私人仓库，丢一些平时的练习、示例代码。另一账号则放上自己觉得比较拿得出手的代码。

现在很多公司技术负责人，也包括一些猎头，开始到github上找人。代码会说话，明眼人自然可以从代码上判断出一个人的水平高低。这也省去了笔试、试用的麻烦，如果我是负责招聘的，也希望通过这种方式，毕竟笔试就写那么几段代码，很难考察出面试者的水平。

另外github上也可以托管博客系统，简单学习一下markdown标记语言，即可开始写作，也是一个选择，它对google搜索引擎比较友好，容易被Google引擎收录。

有的人会说，自己的项目涉密，无法公开，抑或每天996，哪有什么心情写文章、提交开源代码呀？那这时需要搬出第三件法宝。

在公司树立形象

你有没有思考过，你在同事（同学）心目中是怎样的一种形象，技术大牛、独当一面？

现在很多人加入的都是私营企业，人际关系比较淡。有人就会认为，没必要维护良好的关系呀，反正说不定哪天就离开了公司，永不相见。的确，我们不需要像政府机关、公务员那样小心翼翼维护人际关系，我在这儿谈的是一种人设。比如说你在公司独当一面，每次遇到技术难题，都需要你出马解决，那你在同事的心目中，是不是就是技术大牛形象？如果某个同事去了一个更好的公司，碰到有新的机会，会不会第一时间想到你？公司刚进来一个菜鸟，你给予了帮助，对方去了一个新的公司，发现有新的机会，是不是会第一时间就推荐你？

经常可以看到某些文章，说在公司做老好人，没有好报，对其中的观点部分赞同，就是首先做好自己的事情，有余力再考虑帮助他人。如果确实有余力，还是建议尽力帮助寻求帮助的人。特别是对刚进入公司的新人，在环境、技术等还不熟悉的时候，多提供帮助，而不是在那里教训：这也不会，真是笨死了，或者对别人善意的请求置之不理。要知道，这会在别人心目中，留下怎样的一个人设？以后如果有机会，别人会推荐你吗？

一个人的人设可以有多种，你可以尽力维护某（几）种正向的人设，最起码要做到以后别人在聊到你的时候，自然会想到你身上的那个标签，最好结果是有好的机会时，能够第一时间想到你。
