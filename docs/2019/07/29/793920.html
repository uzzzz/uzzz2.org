为什么做了C++开发十几年的老程序员也不敢说自己精通呢？

  • 柚子社区
  • Donate

一个十几年的C++程序员，几乎每天都涉及这门语言，还是觉得远远不到通层面，主要原因是实际工作过程中不可能每个语法细节都能涉及到，而且标准还在不停的更新增加，这门语言是值得研究一辈子，现在很多毕业没有多长的小伙伴，简历上写上精通的基本上面试通过的概率不大：真不知道天高地厚，前几天有个去公司实习的大学生，也是计算机料学与技术专业，让接触这门语言，结果开始搞就傻眼了，这个和学校学的差太远了，学校搞的基本上真皮毛。

这也就是学校和企业最大差距，学校里面主要还是理论灌输为主，说的再直接一点不挂料完成学业为主，企业主要面向社会以结果论，可能在实际操作过程中用的不是最新理念但是功能完成的让客户非常满意，学校学习很多都是理论点到为止，常年战斗在编程第一线的程序员反而掌握的不一定全面，到只要涉及到了，都是实实在在的经验，而且平时工作相对比较忙，有可能跟不上新的标准出来，长此以往时间长了还是觉得这门语言永远没有学完。

这门语言之所以很难精通，主要设计者在理念上想通过一些语法细节约束，做出一门精致的语言，很多业内的大咖对这种设计理念很不爽，编程语言大趋势是走向简单化，从现在比较热门的语言可以看的出，基本上这门语言一旦扎进去就出不来了，而且随着不停的更新语法细节越来越多，同现在编程请言大趋势背道而驰，关键现在很多果均师对这门语言爱不程手，被虐够了反而喜欢上了，一门让人又爱又恨的润程语言，不建议切学者直境选择这门语言作为入门编程语言，容易把自己那点政情给打，现在很多客户端框架设计首选，毕竟既能保证住能又有通向对象设过也只有C++了，实践才是原理，敢不敢用这门语言做个项目，感受下。

大家一起来看看 C++ 的学习之路，从入门到精通都需要掌握什么样的技能。没有全部都掌握的同学，以后可千万别轻易说自己精通 C++ 。

如何才能学好 C++？

这好像是越来越困难的一件事了。

大约十年前在我认真学完一本C++ 之后，我已经觉得自己精通了，但是后来，我才发现当初的我是多么的肤浅。

想要学好并且精通 C++, 不单要学好类等基础知识，还得学会 STL，还要学习 C++1x 中的新特性，并且要适应 C++ 标准的不断变化。
