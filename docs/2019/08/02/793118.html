半个世纪之前，也就是1969年7月16日9时32分，阿波罗11号在美国肯尼迪航天中心发射升空。人类历史上第一次登月之旅的壮举背后，一直助力美国国家航天局(NASA)的IBM大型计算机功不可没。人类社会的进步与发展离不开科技创新这个“后台”，而作为“后台中的后台”，基础架构的与时俱进是永恒的话题。


就在企业翻开“数字化重塑第二篇章”之时，全球企业仍有80%以关键业务为主的应用尚未实现云化。问题的症结究竟在哪里？基础架构这个后台到底够不够硬，能不能在性能、可靠性、安全性等方面消除现存的诸多瓶颈，让企业的云化、AI之旅变得更加顺畅？


基础架构如何现代化？


在任何时候，客户的需求总是第一位的，也是引发变革的最大驱动因素。让我们先看看最具代表性的金融行业的例子。国内一家地区性的中小规模商业银行，其资产规模在1千亿元左右，近三年的IT投入都超过销售额的5%，在3000万元至5000万元之间。这家银行的70多个数据库分布在上百台服务器上，而运维人员只有五六个，同时还要满足新系统开发的需求。再看看它的基础设施，它租赁了电信机房，随着x86平台不断扩展，机房空间的压力、电费压力与日俱增。另外，这家银行虽然拥有很多分布式系统，但是整体利用率不高。


让这家银行焦虑的是，未来其基础构架将向哪个方向发展？最终，IBM协助该银行通过LinuxONE进行了x86整合。LinuxONE“一核一库”的概念，即一个LinuxONE核心接管一个数据库，让用户能够以一个统一的构架承载不同类型的数据库，从而大大节省机房的设备占地空间和电费。这种简单统一的构架、集约的部署方式为用户实现多活的数据中心，以及未来通过容器上云铺设了一条平坦的道路。


IBM 副总裁、大中华区系统部总经理侯淼


IBM 副总裁、大中华区系统部总经理侯淼表示，上面提到的这家银行并不是特例。面对新的业务发展需求，企业的IT架构正进入新一轮变革期。不仅位于北上广深等一线城市的大型银行，就连很多地方性的中小银行也在通过整合、集约化的部署提高基础架构的利用率，提升整体投资回报率。


IDC中国副总裁兼首席分析师武连峰分析认为，人工智能在所有行业不断深入渗透，企业正在进入新的数字化转型阶段。因此，企业需要建立新的数字化转型愿景与路线图，打造以云、智能、安全、开放为核心的未来信息架构。


如今，越来越多的客户在围绕数字化、云和AI实施架构转型。第三方统计数据显示，企业IT系统的重塑将在5年内成规模地集中爆发：2020年，90%的企业将会使用多个云服务和平台，多云环境成为常态；2021年，50%的企业的IT基础架构将采用人工智能提高生产效率、降低管理风险和成本，65%的企业将会针对服务器基础架构部署普遍加密，以保护数据隐私并满足合规要求。2020年，55%的企业将会开启数据中心的现代化建设。


应需而动，IBM推出了现代基础架构（Modern Infrastructure）。以“就绪今日，架构未来”（Ready for Today, Built for Tomorrow）为导向的IBM硬件系统构建的现代基础架构具备三大特点，即负载为要、多云敏捷、安全无虞，它面向AI、大数据、区块链等最严苛需求的工作负载而优化性能，让数据、负载、服务在多云环境中敏捷部署，并以多重数据安全防护保障企业最珍贵数据的安全与合规，从而支撑起企业在AI与多云时代的数字化重塑历程。


企业的基础架构若想满足未来的需要，必须具备以下三个特征：稳态、敏态、智能。稳态是指高可靠、高可用、高安全；敏态是指实现集中简约，并具备敏捷性，灵活扩展；智能则是指满足可分析、可训练、可迭代的要求。“未来，IBM将在基础架构层面投入更多资源和力量，特别是在AI、量子计算、区块链和数据管理等方面。”侯淼表示，“满足负载、多云、安全等需求的新型基础架构，可以帮助客户在数字化重塑的过程中斩获更多。”


谁说主机不能云化？


在过去几十年的企业信息化建设过程中，主机一直是“三高一低”——高可靠、高密度、高安全和低延时的门面担当。如今进入数字化转型第二篇章，主机还能成为企业上云和AI应用的门面担当吗？主机这个后台还能像以前那么“硬气”吗？其实，主机一直在自我进化。IBM大中华区系统部主机及LinuxONE技术总监应康勇表示，IBM主机和LinuxONE已经颠覆了人们以往的印象，成了企业上云的首选。


IBM大中华区系统部主机及LinuxONE技术总监应康勇


2015年8月，IBM推出了LinuxONE，这是IBM在Linux领域中针对企业级计算的主打产品；2016年1月，LinuxONE系列产品在技术特性及开放合作方面取得最新进展，凭借LinuxONE的混合云能力、升级版系统以及持续壮大的生态系统，不同规模的企业得以安全、便捷、高效地进行云应用的开发、部署和管理；2017年9月，IBM发布全新一代产品IBM LinuxONE Emperor II，亮点之一便是对IBM 安全服务容器（IBM Secure Service Container）技术的支持; 2018年4月,IBM发布“云就绪”主机IBM z14 ZR1; 2019年3月，IBM大中华区硬件系统部针对IBM Z与LinuxONE发布三大关键举措,包括IBM z14计算能力随需租用模式、IBM LinuxONE嵌入式解决方案合作模式、LinuxONE三年维保服务。负载、多云、安全，这是现代基础架构的三大核心要素，IBM在主机和LinuxONE上的创新正是围绕这三方面展开的。


IBM主机和LinuxONE一直被认为是最安全的平台。IBM最新一代z14和LinuxONE是IBM Z有史以来出货量最大的一代产品，成功的原因之一就是在安全架构能力上做了大量增强，最具代表性的就是分区的隔离技术、普遍加密技术，以及安全服务容器的专利技术等。企业上云的一个主要障碍是安全性，IBM主机和LinuxONE在安全性方面的持续创新打消了客户关键负载上云的很多顾虑。


通过不断革新，IBM持续升级自身Z以及LinuxONE平台，为企业面向混合多云以及人工智能时代构建了功能强大、高安全性的基础架构组合。主机不仅可以上云，而且可以直接支持混合多云。举例来说，因为LinuxONE本身具有高安全性和高性能，IBM区块链云服务的生产环境就运行在LinuxONE上。


应康勇举了一个客户的成功案例。该客户有一套基于微服务的框架，前台应用部署在x86平台上，中台应用部署在IBM LinuxONE上，后台应用则放在IBM主机上。这样一套“组合拳”实现了跨三个平台的全套应用，可一键式完成运行环境的创建和应用部署，而且完全基于Kubernetes。如果有任何应用的变更，可以通过DevOps技术实现自动化的持续交付。跨三个平台，所有的服务都可以由一个统一的集成服务管理平台进行管理，提供无差别的一致性管理与监控。主机和LinuxONE与云无缝对接，这就是客户要的体验。


应康勇透露说，2019年下半年，IBM将推出z/OS 2.4，它采用容器扩展技术，可以支持更多基于容器化的新应用，不管是基于容器化的分装应用，还是云原生应用，都可以运行在IBM的主机上。有了容器扩展技术，开源社区里的各种新工具、新能力都可以非常容易地集成到主机环境中。开放、创新，主机一样也不缺。对企业来说更具实际意义的是，LinuxONE可以实现纵向、横向双向扩展，在同一个平台上支持稳态业务与敏态业务，为企业关键负载上云奠定更加坚实的基础。


“以前，四五个人管理100台x86设备已经达到极限。今天通过LinuxONE的整合，管理四五百台x86设备也不在话下。通过集中式的处理，LinuxONE把一个平面的停车场变成了立体的停车楼，节少大量的空间和电能，降低了基础架构的整体拥有成本，使管理更加简单。”侯淼表示，LinuxONE最适合x86、云计算和区块链等应用场景，其触角正在向金融、医疗、制造等多行业延伸。


谁能撬动存储底座？


数据是新的生产资料，而人工智能是新的生产力，“数据+智能”将产生颠覆一切的巨大能量。对于企业来说，谁掌握了数据，谁拥有了深刻的洞察，谁将能书写数字化转型最美的第二篇章。


IBM大中华区系统部存储系统总经理吴磊


IBM大中华区系统部存储系统总经理吴磊冷静地分析指出，在人工智能加速创新和应用的当下，企业在数据分析和AI中遇到了一系列数据管理方面的挑战，比如数据摄取和准备的周期太长，多元数据难以聚合，让处在不同基础架构孤岛中的数据流动起来，以及安全地管理和保护数据，并确保可追溯性以及实现全球可访问和协作等。


为了解决上述问题，IBM创新的存储产品可以帮助企业部署一个治理平台，将用于AI模型的数据统一起来，让它们从离散到聚合，从离线到在线，从静止到流动，从而形成端到端的数据管道，实现企业数据随时随地存储，随时随地获取。


数据“后台”是AI应用的底座。不管多么复杂的算法，也不管多么可信的模型，都要基于一个可靠的数据基础，才能最终实现高效的数据流通。IBM提供高效、可靠、开放的现代基础架构，助推AI到企业级、到生产、到产业的“普惠”应用。


今天，全球只有20%的数据可以被搜索及访问，还有80%的数据尚待挖掘，大量数据仍然存储在传统的存储设备中。因此，兼顾传统存储与以软件定义存储、闪存、对象存储等为代表的新兴存储，两手抓、两手硬是对厂商的一个考验。这需要长期的技术积累，以及以客户为中心的持续创新，而这正是IBM存储的优势所在。


IBM存储采用成熟且久经验证的技术，可以灵活部署在公有云与混合云环境中，并以利用企业现有基础设施的特性支持其下一代应用程序的开发，满足软件定义的存储需求。IBM近期发布了FlashSystem 9100全闪存阵列存储系统，它采用IBM Spectrum Virtualize和IBM FlashCore技术，经过NVMe优化，具有灵活、现代、敏捷的特性，同时融合了IBM Spectrum系列产品。另外，IBM云对象存储（IBM Cloud Object Storage）提供面向混合云数据应用需求、可灵活扩展、灵活调度的方案，助力企业实现数据的统一管理、跨多中心应用、链接多种应用接口的能力，以满足不同服务等级，并支持全部存储类型。


消费互联网看的是“颜值”，令人眼花缭乱的各种APP，以及轻松便捷的应用体验。而产业互联网更看重的是“实力”，尤其是在企业上云、AI应用、物联网、5G等快速崛起的今天，打造一个坚固可靠的“后台”是非常必要而且重要的。


聚焦后台，为前台应用提供高效、稳定、可靠的支撑，这是IBM的主动选择，也是其长期积累的优势。如果说以前IBM是张开五指，那么现在则是握指成拳，让基础架构这个数字化转型的底座更加厚实。


企业数字化转型需要一个强有力的后台作为支撑，有后台才能腰杆硬。IBM不就是这个“硬后台”吗？
